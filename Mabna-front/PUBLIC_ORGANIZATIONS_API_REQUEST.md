# درخواست API جدید: لیست عمومی Organizations

## مشکل فعلی
در فرم ثبت‌نام، کاربران جدید نیاز دارند لیست تیم‌های (Organizations) موجود را ببینند تا بتوانند به یک تیم موجود بپیوندند. اما endpoint فعلی `/api/organizations` نیاز به authentication دارد و کاربران جدید که هنوز ثبت‌نام نکرده‌اند، token ندارند.

## راه‌حل پیشنهادی
ایجاد یک endpoint عمومی جدید که بدون نیاز به authentication قابل دسترسی باشد:

### Endpoint جدید

```
GET /api/public/organizations
```

**ویژگی‌ها:**
- بدون نیاز به Authentication
- فقط اطلاعات عمومی و غیرحساس را برمی‌گرداند
- برای استفاده در فرم ثبت‌نام

### Response Format

```json
[
  {
    "id": "uuid-string",
    "name": "نام تیم"
  },
  {
    "id": "uuid-string",
    "name": "نام تیم دیگر"
  }
]
```

**نکات مهم:**
- فقط `id` و `name` برگردانده شود (بدون آمار، تعداد کاربران، یا اطلاعات حساس دیگر)
- این endpoint عمومی است و نباید اطلاعات حساس در آن قرار گیرد
- مرتب‌سازی بر اساس `created_at` یا `name` (اختیاری)

## استفاده در Frontend

Frontend از این endpoint در کامپوننت `Register.jsx` استفاده می‌کند:

```javascript
const response = await fetch(`${API_BASE_URL}/api/public/organizations`);
const organizations = await response.json();
```

سپس لیست تیم‌ها در یک dropdown به کاربر نمایش داده می‌شود تا یکی را انتخاب کند.

## مزایا
1. **امنیت**: اطلاعات حساس در معرض دید عموم قرار نمی‌گیرد
2. **جداسازی**: endpoint عمومی از endpoint احراز هویت شده جدا است
3. **کاربردی**: دقیقاً اطلاعاتی که برای ثبت‌نام نیاز است را فراهم می‌کند

متوسط - این تغییر برای بهبود فرآیند ثبت‌نام ضروری است.
