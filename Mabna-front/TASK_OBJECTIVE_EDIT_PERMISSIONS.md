# قابلیت ویرایش وظایف و اهداف - مجوزهای سازنده

## خلاصه تغییرات

قابلیت ویرایش و حذف وظایف و اهداف اضافه شده است، اما فقط کسی که آن‌ها را ایجاد کرده می‌تواند این عملیات‌ها را انجام دهد.

## تغییرات در وظایف (Tasks)

### 1. ویرایش وظیفه
- **دسترسی:** فقط سازنده وظیفه (`is_creator: true`)
- **مکان:** دکمه "ویرایش وظیفه" در مودال جزئیات وظیفه
- **قابلیت‌ها:**
  - ویرایش عنوان
  - ویرایش توضیحات (با پشتیبانی از mention)
  - تغییر نوع وظیفه (معمولی/ویژه)
  - تغییر وضعیت
  - تغییر مسئول
  - تغییر نتایج کلیدی مرتبط
  - تغییر سررسید

### 2. حذف وظیفه
- **دسترسی:** فقط سازنده وظیفه (`is_creator: true`)
- **مکان:** دکمه "حذف وظیفه" در مودال جزئیات وظیفه
- **تایید:** نیاز به تایید کاربر قبل از حذف

### 3. ویرایش توضیحات
- **دسترسی:** فقط سازنده وظیفه (`is_creator: true`)
- **مکان:** دکمه "✏️ ویرایش توضیحات" در بخش توضیحات
- **قابلیت:** ویرایش inline با MentionTextarea

## تغییرات در اهداف (Objectives)

### 1. ویرایش هدف
- **دسترسی:** فقط سازنده هدف (`is_creator: true`)
- **مکان:** دکمه "ویرایش" در کارت هدف
- **قابلیت‌ها:**
  - ویرایش عنوان
  - ویرایش توضیحات
  - تغییر تاریخ شروع و پایان

### 2. حذف هدف
- **دسترسی:** فقط سازنده هدف (`is_creator: true`)
- **مکان:** دکمه "حذف" در کارت هدف
- **تایید:** نیاز به تایید کاربر قبل از حذف

### 3. مدیریت نتایج کلیدی
- **اضافه کردن:** فقط سازنده هدف (`is_creator: true`)
- **ویرایش:** فقط سازنده هدف (`is_creator: true`)
- **حذف:** فقط سازنده هدف (`is_creator: true`)

## نیازمندی‌های API

برای عملکرد صحیح این قابلیت‌ها، API باید فیلد `is_creator` را برگرداند:

### برای وظایف:
```json
{
  "id": "task-id",
  "title": "عنوان وظیفه",
  "is_creator": true,  // آیا کاربر فعلی سازنده این وظیفه است؟
  // سایر فیلدها...
}
```

### برای اهداف:
```json
{
  "id": "objective-id",
  "title": "عنوان هدف",
  "is_creator": true,  // آیا کاربر فعلی سازنده این هدف است؟
  // سایر فیلدها...
}
```

## API Endpoints مورد استفاده

### وظایف:
- `PUT /api/tasks/:id` - ویرایش کامل وظیفه
- `DELETE /api/tasks/:id` - حذف وظیفه

### اهداف:
- `PUT /api/objectives/:id` - ویرایش هدف
- `DELETE /api/objectives/:id` - حذف هدف
- `PUT /api/keyresults/:id` - ویرایش نتیجه کلیدی
- `DELETE /api/keyresults/:id` - حذف نتیجه کلیدی

## تجربه کاربری

### برای سازنده:
- تمام دکمه‌های ویرایش و حذف نمایش داده می‌شوند
- امکان ویرایش کامل وظایف و اهداف خود

### برای سایر کاربران:
- دکمه‌های ویرایش و حذف مخفی هستند
- فقط امکان مشاهده و تعامل محدود (کامنت، تغییر وضعیت)

## مزایا

1. **امنیت:** فقط سازنده می‌تواند محتوای خود را تغییر دهد
2. **کنترل:** مالکیت واضح روی وظایف و اهداف
3. **انعطاف:** امکان ویرایش کامل برای سازنده
4. **سادگی:** UI واضح و قابل فهم

## نکات پیاده‌سازی

1. همه چک‌های `is_creator` در frontend انجام می‌شود
2. Backend باید validation مناسب داشته باشد
3. در صورت عدم وجود `is_creator`، دکمه‌ها مخفی می‌مانند
4. تمام عملیات ویرایش نیاز به token معتبر دارند
